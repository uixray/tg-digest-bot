## Быстрый старт

Бот уже настроен и запущен через pm2 с автозапуском при загрузке Windows. При обычном использовании ничего дополнительно запускать не нужно — бот работает в фоне.

Чтобы начать работу, откройте своего бота в Telegram и нажмите **Start** или отправьте `/menu`.

## Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Приветствие + главное меню |
| `/menu` | Открыть кнопочное меню |
| `/digest` | Дайджест за последние 24 часа (простой) |
| `/digest ai` | Дайджест за 24 часа с AI-суммаризацией |
| `/digest 24h` | Дайджест за 24 часа |
| `/digest 3d` | Дайджест за 3 дня |
| `/digest 1w` | Дайджест за неделю |
| `/digest 3d ai` | Дайджест за 3 дня + AI |
| `/channels` | Список отслеживаемых каналов |
| `/add @username` | Добавить канал |
| `/remove @username` | Удалить канал из отслеживания |
| `/collect` | Вручную собрать свежие сообщения |
| `/settings` | Текущие настройки |
| `/set cron <выражение>` | Изменить расписание |
| `/set ai <провайдер>` | Сменить AI-провайдер |
| `/help` | Список команд |

### Формат периода

Период указывается в виде `<число><единица>`:
- `h` — часы (например `12h` — 12 часов)
- `d` — дни (например `3d` — 3 дня)
- `w` — недели (например `2w` — 2 недели)

Аргументы можно комбинировать в любом порядке: `/digest 3d ai` или `/digest ai 3d`.

## Управление каналами

### Добавление канала

```
/add @channel_username
```

Бот через userbot-клиент найдёт канал в Telegram, определит его ID и название, и добавит в базу данных. Вы должны быть подписаны на этот канал в аккаунте, через который была выполнена авторизация.

### Удаление канала

```
/remove @channel_username
```

Канал не удаляется из базы физически, а помечается как неактивный (`isActive = false`). Если добавить его снова — он реактивируется вместе со всей историей.

### Просмотр списка

```
/channels
```

Покажет все каналы с пометкой ✅ (активен) или ❌ (неактивен).

## Дайджест

Дайджест — это сводка сообщений из всех активных каналов за указанный период.

### Как формируется

1. **Сбор** — бот через GramJS (userbot) заходит в каждый отслеживаемый канал и загружает последние сообщения (до 50 на канал)
2. **Сохранение** — сообщения сохраняются в PostgreSQL (дубликаты игнорируются)
3. **Выборка** — из базы берутся сообщения за указанный период
4. **Форматирование** — сообщения группируются по каналам и оформляются в HTML

### Простой дайджест

Выглядит так:

```
Дайджест за 5 февраля 2026

Сообщений: 42
━━━━━━━━━━━━━━━━━━
Название канала

  10:30 Текст сообщения (до 150 символов)... 🖼
  11:15 Ещё одно сообщение 🎬
  12:00 🖼 Фото
━━━━━━━━━━━━━━━━━━
Другой канал
  ...
```

### Медиа-индикаторы

Посты с текстом получают эмодзи-бейдж в конце строки:

| Эмодзи | Тип контента |
|--------|-------------|
| 🖼 | Фото |
| 🎬 | Видео |
| 📎 | Документ |
| 🔗 | Ссылка / превью |
| 🎵 | Аудио |
| 🎤 | Голосовое сообщение |
| 😀 | Стикер |
| 📦 | Другое медиа |

Посты без текста (только медиа) отображаются с полной подписью: `🖼 Фото`, `🎬 Видео` и т.д.

### Ссылки

Если у канала есть публичный username — время каждого сообщения будет кликабельной ссылкой на оригинальный пост в Telegram.

## AI-суммаризация

При запросе дайджеста с флагом `ai` (кнопка «🤖 Дайджест + AI» или команда `/digest ai`) бот дополнительно отправляет тексты сообщений в нейросеть для создания краткой структурированной сводки.

### Поддерживаемые провайдеры

#### LM Studio (локальная модель)

- Работает на вашем компьютере, бесплатно
- Совместим с OpenAI API (`http://localhost:1234/v1`)
- Нужно запустить LM Studio и загрузить модель перед использованием
- Настройка в `.env`: `AI_PROVIDER=lmstudio`

#### YandexGPT (облачный)

- Облачный сервис Яндекса, требует API-ключ
- Нужен `YANDEX_FOLDER_ID` и `YANDEX_API_KEY` в `.env`
- Настройка: `AI_PROVIDER=yandexgpt`

#### Отключение AI

- `AI_PROVIDER=none` — AI-суммаризация недоступна, кнопка «Дайджест + AI» покажет только простой дайджест

### Переключение провайдера

Через бота:
```
/set ai lmstudio
/set ai yandexgpt
/set ai none
```

Настройка сохраняется в базе данных и применяется сразу, без перезапуска.

### Как работает суммаризация

1. Тексты всех сообщений объединяются (до 15 000 символов)
2. К сообщениям с медиа добавляется пометка: `[Прикреплено: фото/видео/...]`
3. Нейросети передаётся системный промпт с инструкциями:
   - Выделить основные темы и события
   - Дать краткое описание каждой темы (1-2 предложения)
   - Объединить связанные новости из разных каналов
   - Упомянуть наличие медиаконтента

Результат AI-сводки выводится в конце дайджеста в блоке **«AI-итоги»**.

## Автоматический дайджест по расписанию

Бот автоматически отправляет дайджест по расписанию cron. По умолчанию — **каждый день в 9:00 по московскому времени**.

Расписание задаётся cron-выражением:

| Выражение | Расписание |
|-----------|-----------|
| `0 9 * * *` | Каждый день в 9:00 (по умолчанию) |
| `0 9,18 * * *` | В 9:00 и 18:00 каждый день |
| `0 9 * * 1-5` | В 9:00 по будням (пн-пт) |
| `0 9 * * 1` | В 9:00 каждый понедельник |
| `*/30 * * * *` | Каждые 30 минут |

### Изменение расписания

```
/set cron 0 9,18 * * *
```

Настройка сохраняется в базе данных. Для применения нового расписания нужно перезапустить бота (кликнуть `restart.bat`).

### Что входит в автоматический дайджест

- Сообщения за последние 24 часа
- AI-суммаризация включается, если `AI_PROVIDER` не равен `none`
- Дайджест отправляется владельцу бота (по `OWNER_CHAT_ID`)

## Настройки

Команда `/settings` показывает текущую конфигурацию:

- **Расписание** — cron-выражение
- **Часовой пояс** — для cron (по умолчанию `Europe/Moscow`)
- **AI-провайдер** — `lmstudio` / `yandexgpt` / `none`
- **Активных каналов** — количество

### Команда `/set`

```
/set cron <выражение>   — изменить расписание
/set ai <провайдер>     — сменить AI (yandexgpt / lmstudio / none)
```

Изменения сохраняются в таблице `Settings` в базе данных и имеют приоритет над значениями из `.env`.

## Архитектура проекта

Бот состоит из четырёх основных подсистем:

```
┌─────────────────────────────────────────────────┐
│                  Telegram                        │
│                                                  │
│   Вы (владелец)  ←──→  Telegraf Bot              │
│                         (Bot API)                │
│                                                  │
│   Каналы  ←────────→  GramJS Userbot             │
│                        (MTProto)                 │
└────────────────────┬────────────────────────────┘
                     │
        ┌────────────┼────────────┐
        │            │            │
        ▼            ▼            ▼
  ┌──────────┐ ┌──────────┐ ┌──────────┐
  │PostgreSQL│ │ Дайджест  │ │   AI     │
  │ (Prisma) │ │ Builder + │ │ LMStudio │
  │          │ │ Formatter │ │ YandexGPT│
  └──────────┘ └──────────┘ └──────────┘
```

### 1. Telegraf Bot (Bot API)

Обычный Telegram-бот, созданный через @BotFather. Отвечает на команды и кнопки. Используется для взаимодействия с вами — отправляет дайджесты, принимает команды.

Доступ ограничен: бот отвечает **только владельцу** (по `OWNER_CHAT_ID`). Все остальные сообщения молча игнорируются.

### 2. GramJS Userbot (MTProto)

Подключается к Telegram как ваш аккаунт (через сессию, полученную при авторизации `npm run auth`). Используется для чтения сообщений из каналов — Bot API не позволяет читать каналы, на которые подписан пользователь.

**Важно:** userbot работает от имени вашего аккаунта. Он может читать только те каналы, на которые вы подписаны.

### 3. PostgreSQL + Prisma ORM

Все собранные сообщения и метаданные хранятся в базе PostgreSQL. Prisma обеспечивает типизированный доступ к данным. Это позволяет:
- Не загружать одни и те же сообщения повторно
- Формировать дайджесты за любой период из уже собранных данных
- Хранить историю всех дайджестов

### 4. AI-провайдеры

Опциональный модуль. Отправляет собранные тексты нейросети для создания краткой сводки. Поддерживает два провайдера через единый интерфейс `AIProvider` — можно легко добавить новые.

### 5. Cron-планировщик

Модуль `node-cron` запускает автоматическое формирование и отправку дайджеста по расписанию. Cron-выражение читается из базы данных (таблица `Settings`) или из `.env`.

## База данных

### Модели (таблицы)

#### Channel — отслеживаемые каналы

| Поле | Тип | Описание |
|------|-----|---------|
| id | Int | Внутренний ID |
| telegramId | BigInt | Telegram ID канала |
| username | String? | Username канала (@channel) |
| title | String | Название канала |
| isActive | Boolean | Активен ли канал |
| addedAt | DateTime | Дата добавления |

#### Message — собранные сообщения

| Поле | Тип | Описание |
|------|-----|---------|
| id | Int | Внутренний ID |
| telegramId | Int | ID сообщения в Telegram |
| channelId | Int | FK → Channel |
| text | String? | Текст сообщения |
| mediaType | String? | Тип медиа (photo, video, document...) |
| url | String? | Ссылка на пост в Telegram |
| date | DateTime | Дата публикации |
| collectedAt | DateTime | Когда собрано |

Уникальное ограничение: `(channelId, telegramId)` — одно сообщение из канала не может быть сохранено дважды.

#### Digest — сформированные дайджесты

| Поле | Тип | Описание |
|------|-----|---------|
| id | Int | Внутренний ID |
| content | String | Текст дайджеста |
| aiSummary | String? | AI-сводка (если запрашивалась) |
| periodStart | DateTime | Начало периода |
| periodEnd | DateTime | Конец периода |
| createdAt | DateTime | Дата создания |

#### DigestMessage — связь дайджестов с сообщениями

Связующая таблица: какие сообщения вошли в какой дайджест.

#### Settings — настройки

| Ключ | Описание |
|------|---------|
| `digest_cron` | Cron-выражение расписания |
| `ai_provider` | Активный AI-провайдер |

## Как это работает внутри

### Процесс запуска (src/index.ts)

1. Подключение к PostgreSQL через Prisma
2. Подключение GramJS userbot к Telegram (через сохранённую сессию)
3. Регистрация команд бота в Telegram UI (меню команд)
4. Запуск Telegraf-бота (long polling)
5. Запуск cron-планировщика

### Процесс формирования дайджеста

```
Пользователь нажимает "📰 Дайджест"
        │
        ▼
  collectAll() — GramJS обходит все каналы
        │          загружает до 50 последних сообщений
        │          сохраняет в БД (upsert, без дубликатов)
        │          пауза 1.5 сек между каналами (rate limit)
        │
        ▼
  buildDigest() — запрос к БД: сообщения за период
        │          группировка по каналам
        │          [опционально] отправка в AI для суммаризации
        │          сохранение дайджеста в БД
        │
        ▼
  formatDigest() — HTML-форматирование
        │           медиа-эмодзи, ссылки, время
        │
        ▼
  splitMessage() — разбивка на части ≤ 4096 символов
        │            (лимит Telegram API)
        │
        ▼
  Отправка пользователю через Telegraf
```

### Безопасность

- **Доступ ограничен**: middleware `ownerOnly` проверяет `ctx.from.id === OWNER_CHAT_ID` на каждом запросе. Чужие сообщения молча отбрасываются.
- **Секреты в .env**: токены и ключи не хранятся в коде.
- **Userbot-сессия**: строка `TELEGRAM_SESSION` — это ваш ключ доступа к Telegram-аккаунту. Не передавайте её третьим лицам.

## Решение проблем

### Бот не отвечает

1. Запустите `status.bat` — проверьте, что статус `online`
2. Запустите `logs.bat` — посмотрите на ошибки
3. Попробуйте `restart.bat`

### «Нет новых сообщений»

- Убедитесь, что каналы добавлены: `/channels`
- Проверьте, что за указанный период действительно были посты
- Попробуйте `/collect` — ручной сбор сообщений
- Попробуйте больший период: `/digest 3d` или `/digest 1w`

### AI-суммаризация не работает

**LM Studio:**
- Убедитесь, что LM Studio запущен и модель загружена
- Проверьте, что сервер доступен: `http://localhost:1234/v1`

**YandexGPT:**
- Проверьте `YANDEX_FOLDER_ID` и `YANDEX_API_KEY` в `.env`
- Убедитесь, что ключ действителен и не истёк

### Канал не добавляется

- Проверьте, что вы подписаны на канал в аккаунте, который был авторизован
- Убедитесь, что username указан правильно: `/add @channel_name`

### После изменений в коде ничего не изменилось

Запустите `build-and-restart.bat` — он перекомпилирует TypeScript в JavaScript и перезапустит бота.

### Нужно переавторизовать userbot

```bash
npm run auth
```

Скрипт запросит номер телефона и код из Telegram. После успешной авторизации скопируйте строку сессии в `.env` (переменная `TELEGRAM_SESSION`) и перезапустите бота.
